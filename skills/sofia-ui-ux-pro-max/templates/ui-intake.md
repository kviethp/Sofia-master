# UI Intake Template

Use this template at the start of a UI/UX task.

## Context Summary
- **Task type:** landing-page | dashboard | admin | web-app | mobile-app | component | design-system | ui-audit
- **Product / feature:**
- **Primary user:**
- **Primary goal:** conversion | activation | trust | clarity | retention | engagement | task success
- **Platform:** web | mobile | responsive | desktop
- **Stack:**
- **Brand tone:** trusted | premium | playful | technical | minimal | bold
- **Data density:** low | medium | high
- **Accessibility priority:** normal | high | critical
- **Constraints:** mobile-first, dark mode, delivery speed, existing design system, legacy UI, etc.

## What to infer if missing
- product type
- trust level needed
- hierarchy needs
- likely CTA pattern
- likely state complexity
- likely responsive risk areas

## Minimal follow-up questions
Ask only if the answer materially changes the design direction:
1. What is the product / feature and who is it for?
2. Is the main goal conversion, trust, clarity, or dense productivity?
3. What stack or component system is in use?
4. Is there an existing brand or design system to preserve?

## Routing decision
If the work changes how the UI looks, feels, moves, or is interacted with, continue with the Sofia UI UX Pro Max flow.

## Next step
Run design-system generation unless direction is already documented.
