# Sofia UI Routing Decision Rubric

Route to the Sofia UI UX Pro Max flow when the request primarily affects:
- visual structure
- layout or hierarchy
- typography or color
- responsive behavior
- component states
- navigation clarity
- touch/interaction quality
- accessibility or usability
- design system consistency
- perceived polish or trust

## Strong triggers
- "design", "redesign", "polish", "make it look better"
- landing page, dashboard, admin, onboarding, checkout, settings
- button, modal, form, table, card, chart, nav, sidebar
- responsive, accessibility, hierarchy, spacing, typography, color, motion

## Do not route here for
- pure backend work
- API/data modeling only
- infra/devops only
- debugging unrelated to interface behavior

## Default action
If unsure but the task touches both frontend behavior and UI quality, route here first, establish the design direction, then continue implementation.
